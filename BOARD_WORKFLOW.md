# Board Workflow

We use GitHub Projects as the main task board.

## Columns

### Backlog

All tasks that may be worked on in the future.

### Ready

Tasks selected for the current weekly iteration.

### In Progress

Tasks currently being worked on.

### In Review

Tasks ready for review or verification.

### Done

Completed tasks.

## Rules

- Add new ideas and future work to **Backlog**
- Move tasks to **Ready** during weekly planning
- Move a task to **In Progress** when work starts
- Move a task to **In Review** when it is ready to check
- Move a task to **Done** when it is completed
- Unfinished tasks may remain in **Ready** or be re-evaluated in the next sync
