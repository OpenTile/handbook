# Collaboration Handbook

This repository documents how we work together: process, planning cadence, task management, and weekly goals.

## Purpose

The goal is to make the workflow explicit, lightweight, and easy for new collaborators to understand.

## Working model

We use:

- weekly iterations
- asynchronous daily updates via audio messages
- GitHub Projects for task tracking
- markdown documentation in English

## Weekly iteration structure

Each weekly sync includes:

- demo of completed work
- retrospective
- planning for the next week
- project priority review

## Daily updates

Daily updates are shared asynchronously in the group chat.

Each update should include:

- what was done
- what will be done next
- open questions
- blockers

## Task board

We use the following task states:

- **Backlog** - all future tasks
- **Ready** - tasks selected for the current iteration
- **In Progress** - tasks being worked on
- **In Review** - tasks awaiting review
- **Done** - completed tasks
